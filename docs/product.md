# 产品定义 - Orbitchat

## 项目愿景

Orbitchat 是一个学习型社交平台项目，目标是通过构建完整的、生产级别的社交应用，学习现代全栈 TypeScript 开发和 AI 协作开发（Agentic Engineering）。

本项目不是一次性产品，而是一个**长期演进的学习项目**，会随着技术深入和业务理解逐步迭代。

---

## 产品核心目标

### 长期目标（完整社交平台）

最终目标是构建一个具有以下功能的社交平台：

| 功能模块 | 描述 | 优先级 |
|---------|------|--------|
| **用户系统** | 注册、登录、个人资料 | P0 |
| **社交关系** | 关注/粉丝、推荐、搜索 | P1 |
| **内容系统** | 发布、编辑、删除动态 | P1 |
| **聊天能力** | 一对一和群组聊天、消息推送 | P1 |
| **通知系统** | 关注、评论、点赞、聊天通知 | P1 |
| **多媒体** | 图片、视频上传和处理 | P2 |
| **个人橱窗** | 个人主页、动态列表展示 | P1 |
| **数据分析** | 用户行为分析、内容热度 | P3 |

### 当前阶段目标（Phase 0 - Foundation）

**当前阶段的唯一目标是完成工程搭建和开发规范建设。**

✅ **可以做**：
- 初始化完整的 monorepo 结构
- 创建后端和前端基础框架
- 配置开发工具链
- 编写架构文档
- 创建开发规范

❌ **不要做**：
- 实现用户认证系统
- 创建数据库表
- 实现任何业务逻辑
- 集成第三方服务

## 技术栈决策

### 为什么选择这个技术栈？

#### 后端：Bun + Hono

```
Bun - 现代 JavaScript/TypeScript 运行时
  ✓ 性能更高（JIT 编译）
  ✓ 内置 TypeScript 支持（零配置）
  ✓ 内置 SQLite 支持
  ✓ 简单、现代的 API

Hono - 极轻量级 Web 框架
  ✓ 仅 15KB 大小
  ✓ 完美的 TypeScript 支持
  ✓ 兼容 Bun、Node.js、Deno
  ✓ 学习曲线平缓
```

**不选 Express 的原因**：
- 太重，社区库选择太多容易迷茫
- 当前学习项目需要精简

**不选微服务的原因**：
- 项目初期不需要
- 后续可演进到微服务架构

#### 前端：Next.js 14+

```
Next.js App Router
  ✓ 现代的 React 开发体验
  ✓ 内置 API Routes（开发方便）
  ✓ 自动代码分割和优化
  ✓ 完美的 TypeScript 支持
  ✓ 未来前移成本低
```

#### Monorepo：pnpm

```
pnpm workspace
  ✓ 快速高效（基于内容寻址）
  ✓ 严格的依赖隔离
  ✓ 可并行运行脚本
  ✓ 磁盘空间节省
```

## 开发遵循原则

### 1. 文档驱动

所有功能从文档开始：
1. 编写功能设计文档
2. 编写 API 规范
3. 编写代码
4. 更新相关文档

### 2. 类型安全优先

- TypeScript strict mode
- 禁止使用 `any`
- 完整的类型定义

### 3. AI Agent 友好

项目结构清晰便于 AI 理解和维护：
- 明确的目录组织
- 清晰的命名规范
- 完整的文档

### 4. 渐进式复杂度

避免过度设计：
- 当前不需要的功能暂不实现
- 按需添加复杂度
- 保持项目简洁易维护
